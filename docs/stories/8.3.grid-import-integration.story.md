# Story 8.3: Grid Import Integration in Grids Page

## Status

Draft

## Story

**As a** teacher,
**I want** an "Import CSV" button on the grids page alongside the existing "Create New Grid" button,
**so that** I can easily access the import feature from the same place I manage my grids.

## Acceptance Criteria

1. An "Importer CSV" button appears next to the existing "Create New Grid" button on `app/grids/page.tsx`
2. Clicking the button opens the `GridCSVImporter` modal from Story 8.2
3. After a successful import, the new grid appears immediately in the grid listing (via existing `useLiveQuery`)
4. The "Create New Grid" button and link to `/grids/new` remain unchanged
5. The empty state (no grids) also shows the import option as an alternative to creating manually
6. Both buttons are visually balanced and tablet-friendly (min 44px touch targets)
7. The existing grid deletion functionality is not affected

### Integration Approach

Minimal changes to `page.tsx`:
1. Add one `useState` for modal visibility
2. Add one button in the header `flex` container
3. Add one conditional render for the modal
4. Add one button in empty state

The `useLiveQuery(() => db.grids.toArray())` already powers the listing — any `db.grids.add()` from the importer will automatically trigger a re-render. No additional plumbing needed.
