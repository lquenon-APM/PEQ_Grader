# Story 8.1: Parseur CSV pour Grilles d'Évaluation

## User Story

**As a** Teacher
**I want to** have a reliable CSV parser that transforms my grid file into a valid Grid object
**So that** I can prepare my grading grids in a spreadsheet and import them without manual re-entry.

## Story Context

**Existing System Integration:**
- Integrates with: `lib/db.ts` (Interface `Grid` — structure des compétences/indicateurs)
- Follows pattern: `components/students/CSVImporter.tsx` (parsing CSV existant pour les élèves)
- Technology: TypeScript, Jest for testing
- No new dependencies — pure functions only (`String.split`, `crypto.randomUUID`)

## Acceptance Criteria

**Functional Requirements:**
1. Le parseur accepte du texte CSV avec `;` ou `,` comme séparateur (auto-détecté).
2. Le parseur supporte une ligne d'en-tête optionnelle (`Competency;Indicator;Critical`) qui est ignorée si détectée.
3. Les lignes avec le même nom de compétence sont regroupées en une seule compétence avec plusieurs indicateurs.
4. Le champ `critical` de chaque indicateur est parsé depuis le CSV (`true`/`false`, `1`/`0`, `oui`/`non`, `yes`/`no` — insensible à la casse).
5. Les erreurs de validation sont retournées par ligne avec le numéro de ligne, permettant à l'UI de signaler les problèmes.
6. Un label de compétence vide ou un texte d'indicateur vide produit une erreur de validation pour cette ligne.
7. Un CSV entièrement valide retourne un objet `Grid["structure"]` conforme à l'interface existante dans `lib/db.ts`.
8. Le parseur retourne un résultat structuré : `{ gridStructure, errors[], validLineCount, invalidLineCount }`.

## Technical Notes

- **Module créé :** `lib/grid-csv-parser.ts` — module dédié car la logique est plus complexe que le `CSVImporter` existant (regroupement, multi-format)
- **Interfaces exportées :** `GridParseError` (lineNumber, line, message) et `ParsedGridResult` (gridStructure, errors, validLineCount, invalidLineCount)
- **Fonction principale :** `parseGridCSV(text: string, options?: { separator?: string }): ParsedGridResult`
- **Auto-détection du séparateur :** Comptage des occurrences de `;` vs `,` sur la première ligne non vide
- **Détection d'en-tête :** Match case-insensitive contre les patterns connus (FR : compétence/indicateur/critique, EN : competency/indicator/critical)
- **Regroupement :** `Map<string, Competency>` pour préserver l'ordre d'insertion
- **Gestion des fins de ligne :** Normalisation `\r\n` → `\n` pour compatibilité Windows/Mac/Linux

### Data Model Reference

```typescript
// Output conforme à Grid["structure"] de lib/db.ts
structure: {
  competencies: Array<{
    id: string;        // UUID généré par crypto.randomUUID()
    label: string;     // Colonne "Compétence" du CSV
    indicators: Array<{
      id: string;      // UUID généré par crypto.randomUUID()
      text: string;    // Colonne "Indicateur" du CSV
      critical: boolean; // Colonne "Critique" du CSV
    }>;
  }>;
};
```
