# Story 8.2: Grid CSV Importer UI Component

## Status

Done

## Story

**As a** teacher,
**I want** a visual import interface that previews my CSV file and shows validation feedback,
**so that** I can verify my grid data before importing and correct any errors in my file.

## Acceptance Criteria

1. A modal component `GridCSVImporter` opens with a file upload zone (drag-and-drop style, `.csv` files only)
2. After file selection, the CSV is parsed using `parseGridCSV` from Story 8.1 and a preview table is displayed
3. The preview table shows: line number, competency, indicator text, critical flag, and valid/invalid status per row
4. A summary bar shows count of valid and invalid lines (same pattern as `CSVImporter.tsx`)
5. A text input field allows the user to set the grid title (required before import)
6. Invalid lines are highlighted in red (`bg-rose-50`) and valid lines are unmarked
7. The "Import" button is disabled until: at least 1 valid line AND grid title is filled
8. On import: a `Grid` object is created with `version: 1` and saved via `db.grids.add()`
9. After successful import, the modal closes and the user is returned to the grids listing
10. The user can select a different file to re-parse without closing the modal
11. Touch targets are minimum 44x44px; all interactive elements are tablet-friendly

## QA Results

### Review Date: 2026-03-09

### Reviewed By: Quinn (Test Architect)

### Code Quality Assessment

**Overall: GOOD** — Clean implementation that closely follows the established `CSVImporter.tsx` pattern. All 11 acceptance criteria are addressed. Code is well-structured with clear separation between parsing (delegated to `parseGridCSV`) and UI concerns.

**Strengths:**
- Follows existing CSVImporter.tsx pattern faithfully (modal structure, badges, table layout)
- Proper error handling: import failure shows error message, modal stays open
- `buildPreviewRows()` helper cleanly bridges the gap between parser output (grouped structure) and per-line preview display
- Touch targets consistently use `min-h-[48px]` (exceeds 44px requirement)
- Import button correctly disabled until both conditions met (valid lines + title)
- French UI labels consistent with the rest of the app

**Concerns (Medium):**
- **Duplicated logic**: `detectSeparator()` and `HEADER_PATTERNS` are duplicated from `grid-csv-parser.ts`. If the parser changes separator detection or header patterns, this component will fall out of sync. Consider exporting these from the parser module.
- **No file input reset**: When choosing a different file with the same name, the browser's `onChange` may not fire because the input value hasn't changed. The `<input>` element's value should be reset after each read.

**Concerns (Low):**
- `previewRows.length === 0` is used as the "no file uploaded" indicator, but a valid CSV with zero data lines would also show the upload zone instead of an empty table. Edge case, unlikely in practice.
